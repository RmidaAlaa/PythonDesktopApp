Getting Started

AWG Kumulus Device Manager is a desktop application focused on managing STM32 devices: detecting boards, loading/reading UIDs, flashing firmware, generating reports, and exporting/sending results (Email/OneDrive). This manual provides detailed instructions for all features and configurations.

System Requirements

Minimum Requirements

  • Operating System: Windows 10/11 (x64), Linux, or macOS
  • Disk Space: ~200 MB for application, plus space for logs and reports
  • Memory: 512 MB RAM minimum
  • USB Port: For connecting embedded devices
  • Internet Connection: Required for first-run setup, firmware downloads, and email features

Optional Requirements

  • SMTP Credentials: For email report sending
  • OneDrive Account: For cloud storage integration
  • STM32CubeIDE: For STM32 project development (optional)
  • Python 3.11+: For development/source builds

Reading Device UID

To read the microcontroller UID:

  1. Select the device row to view its details in the right panel.
  2. To load the UID onto the board (when required), check the left checkbox ‚ÄúCheck to load UID from board‚Äù. This flashes the UID helper firmware (GetMachineUid.bin) and refreshes device information.
  3. Click Read UID to open Device UID & Info. For STM32 devices, this dialog can also include a Read UID button.
Auto-flash: When enabled, the app can automatically flash the UID helper firmware when a supported device connects.

Manual Device Detection

Step 1: Connect your device via USB cable
Step 2: Click the "Refresh Devices" button in the device panel
Step 3: Wait for the scan to complete (usually 2-5 seconds)
Step 4: Detected devices will appear in the device table

Automatic Device Monitoring

The application includes real-time device monitoring that:

  • Automatically detects when devices are connected (hotplug)
  • Detects when devices are disconnected
  • Updates the device list every 5 seconds
  • Maintains device history across sessions
  • Tracks connection count and last seen timestamps
Note: Real-time monitoring is enabled by default. You can see connection/disconnection events in the log area.

Device Information Display

For each detected device, the following information is displayed:

  • Port: Serial port identifier (e.g., COM3, /dev/ttyUSB0)
  • Type: Board type (STM32 or Unknown)
  • VID:PID: Vendor ID and Product ID in hexadecimal
  • Status: Connection status (Connected/Disconnected)
  • Health: Health score (0-100) based on connection stability and information completeness
  • Name: Custom name or default name
  • Last Seen: Timestamp of last detection

Troubleshooting

Device Detection Issues

Problem: No devices detected

Solutions:

  1. Verify USB cable is properly connected
  2. Check that device is powered on
  3. Ensure appropriate drivers are installed:
    • STM32: STM32 Virtual COM Port drivers
  4. On Linux, add user to dialout group: sudo usermod -a -G dialout $USER
  5. Try clicking "Refresh Devices" manually
  6. Check Device Manager (Windows) or lsusb (Linux) to verify device is recognized by OS

Firmware Flashing Issues

Problem: Firmware flashing fails

Solutions:

  1. Ensure device is in bootloader mode (if required):
    • STM32: Enter DFU mode (varies by board)
  2. Verify firmware file format is correct (.bin or .elf)
  3. Check that firmware is compatible with device board type
  4. Try a different USB port
  5. Verify helper tools are installed (check logs)
  6. Check device is not in use by another application
  7. Review log output for specific error messages

Email Sending Issues

Problem: Email not sending

Solutions:

  1. Verify SMTP settings are correct:
    • Host and port match your email provider
    • TLS/SSL settings are correct
  2. Check credentials are stored in keyring:
    • Windows: Credential Manager
    • macOS: Keychain Access
    • Linux: Secret Service
  3. For Gmail, use app-specific password (not regular password)
  4. Verify network connection is active (emails will be queued if offline)
  5. Check firewall settings aren't blocking SMTP
  6. Review log output for specific error messages

OneDrive Integration Issues

Problem: OneDrive sync not working

Solutions:

  1. Verify OneDrive folder path is correct and accessible
  2. Check that you have write permissions to the folder
  3. Ensure OneDrive is running and synced
  4. Test connection using "Test Connection" button in settings
  5. Verify user folder name is set correctly
  6. Check that auto-create folders is enabled if needed

STM32CubeIDE Issues

Problem: STM32CubeIDE not found

Solutions:

  1. Install STM32CubeIDE from STMicroelectronics website
  2. Set STM32CUBEIDE_BIN environment variable to executable path
  3. Or set STM32CUBEIDE_HOME to installation directory
  4. Verify installation by running STM32CubeIDE manually
  5. Check Windows Registry if on Windows (for automatic detection)

Application Update Issues

Problem: Update download fails or installer doesn't start

Solutions:

  1. Check your internet connection
  2. If download fails, try clicking "Install Update" again
  3. If the installer doesn't launch, check your "Downloads" folder and run the installer manually
  4. Ensure you have sufficient disk space
  5. If issues persist, download the latest version directly from GitHub

Application Crashes

Problem: Application crashes or freezes

Solutions:

  1. Check log files in app data directory for error messages
  2. Verify all required Python packages are installed
  3. Try deleting configuration file and restarting (backup first!)
  4. Check available disk space
  5. Verify system requirements are met
  6. Contact support with log files

Log Files

Log files are stored in:

  • Windows: %APPDATA%\AWG-Kumulus\logs\
  • Linux: ~/.local/share/awg-kumulus/logs/
  • macOS: ~/Library/Application Support/AWG-Kumulus/logs/

Log files are named with timestamps and can be attached when contacting support.
